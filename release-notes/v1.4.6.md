## v1.4.6: fix mobile header in the installed app

Fixes the chopped-up top bar seen only in the installed (Add to Home Screen) app, where the
header buttons and drop shadow spilled out of the header background and looked broken.

- **Header now accounts for the status-bar safe area.** The mobile header pads itself below the
  status bar with `env(safe-area-inset-top)`, but its height was fixed at `h-16`. In a Safari tab
  the inset is `0` so it fit; in the standalone home-screen app the ~50px inset was consumed
  inside that fixed height, pushing the controls and shadow past the solid background. The header
  now uses a minimum height so it grows to fit the inset plus the controls. Desktop is unchanged.

This was the real cause of what looked like a stale/old header in the installed app; it was a
standalone-only layout issue, not caching.
