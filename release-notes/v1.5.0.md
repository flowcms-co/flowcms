## v1.5.0: workspace menu layering fix

A follow-up to v1.4.9. That release tried to float the workspace switcher menu above the page
content but only lifted an inner wrapper, which wasn't enough, so on desktop the menu could still
slip behind the content where the two overlapped.

- **Workspace switcher now floats correctly.** Opening the workspace menu (top of the sidebar)
  layers cleanly above the page content and top bar, as a popover should. The fix lifts the whole
  sidebar column's stacking context rather than just the menu, so nothing on the page can paint
  over it. No layout or behavior change otherwise.

Studio-only, a stacking-order (z-index) fix; no API changes. If you updated to v1.4.9 for this,
update to v1.5.0 to actually get it.
