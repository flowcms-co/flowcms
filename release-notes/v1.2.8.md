## Highlights

A naming fix for the reusable components work in v1.2.7. The field for grouping fields is called "Component" again, because it can either be a one-off group defined right on the type (Inline) or a reference to a Reusable component from the library; only the latter is actually reusable.

## Fixes and improvements

- Clearer "Component" field naming: the field type and its "Add component" button no longer say "Reusable component", since that field can be an inline (one-off) group as well as a reference to a library component. The Reusable components tab and the reference note (shown only when a field points at a library component) keep their wording, and the editor now labels each field as "component" or "reusable component" based on how it is set up.
- General stability and performance improvements.
