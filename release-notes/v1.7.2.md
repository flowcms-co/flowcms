# v1.7.2: consent done properly, and managed updates that look the part

## Updating on Railway and Render now feels like updating anywhere else

One-click platform updates (added in v1.7.0) used to spin a small icon on
the button while the rollout happened. They now run the same full-screen,
locked upgrade experience compose installs get: a live step tracker
(requesting the deploy, the platform building, restarting and verifying),
a clear "please don't refresh this page" notice, and an automatic reload
onto the new version. Reload the tab mid-rollout and the overlay comes
back on its own and keeps tracking.

## Consent, captured and evidenced

- **Setup asks properly.** The first-run screen now requires two explicit,
  unticked-by-default checkboxes before the admin account can be created:
  acceptance of the Terms of Service and Privacy Policy (with essential
  service and security emails), and the product-email opt-in. Terms open
  on flowcms.co in a new tab. The API enforces both server side.
- **Existing accounts are asked once.** Anyone whose account predates
  consent capture gets a one-time prompt on their next visit with the same
  two checkboxes, and continues right after accepting.
- **Every acceptance is evidenced.** An append-only consent record stores
  when it happened, the request IP the server saw, the public IP the
  browser reported for itself, and the parsed browser, OS and device.
  Evidence writing is fail-soft and can never block the acceptance itself.

## Public signup is gone

Open registration has been disabled by default since the security
hardening in June, but the login screen still offered "Create an account"
and let people fill a form that could only be refused. The page, the
link and the API endpoint have been removed; accounts come from the
first-run claim or an admin invite, and the old /signup URL simply lands
on sign-in.

## Also fixed

- The assets library no longer shows a stray lowercase "all" folder chip
  next to "All assets" in workspaces with unfoldered media.
