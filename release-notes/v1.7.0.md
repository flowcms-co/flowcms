# v1.7.0: updates from inside the CMS, everywhere

The rule for Flow CMS updates is simple: nobody should ever need a terminal.
Compose self-hosts got there in v1.6.3 when the updater started syncing deploy
configs. This release finishes the job for managed platforms and removes the
last known ways an install could demand manual attention.

## One-click updates on Railway and Render

Managed platforms own the container lifecycle, so the all-in-one image can't
replace itself — but every platform has a supported way to trigger a redeploy
remotely, and now the studio uses it.

- **Set it up once, in the app.** Settings → System → Updates now takes a
  Railway API token or a Render Deploy Hook URL, stored encrypted at rest like
  every other credential. Operators who prefer env vars can set
  `RAILWAY_API_TOKEN` or `RENDER_DEPLOY_HOOK_URL` instead.
- **Then updating is one button.** "Update to vX.Y.Z" asks the platform to
  redeploy: Railway through its API (the service and environment ids come from
  Railway's own injected variables), Render through the deploy hook. The
  platform pulls the newest image, migrations apply on boot, and the studio
  reloads itself when the new version is live.
- The manual dashboard walkthrough remains as the fallback until a credential
  is saved, and you can disconnect at any time.

## Upgrades apply the release's deploy config first

`docker-compose.prod.yml` joins the Caddyfile as a release-managed file on
compose self-hosts, and config sync now runs before the image pull, so the
whole upgrade executes against the new release's compose file and routes.
Customized files are still never touched.

## A storage problem can no longer take the CMS down

An unwritable media directory used to crash the API at boot in a restart loop —
a full outage over a filesystem path. Now:

- if S3 is fully configured, the API falls back to it automatically with a
  clear warning in the logs
- otherwise the CMS boots and runs normally, and only uploads fail, with a
  message that says exactly what to fix
