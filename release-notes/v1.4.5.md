## v1.4.5: self-healing updates for the installed app

A follow-up to the v1.4.3/v1.4.4 PWA work. The `no-store` shell header stops new stale copies
from being cached, but an iOS home-screen web app keeps its own private cache that "Clear Website
Data" never clears, so an already-stale shell could keep loading after a deploy. This adds a safety
net so the installed app heals itself.

- **Build-version gate.** Each build stamps a per-deploy id into the bundle. A small client check
  asks the server which build it is running (a no-store `/api/app-version` route) on launch and
  whenever the app returns to the foreground; if the loaded shell is an older build than the
  server, it reloads once to pull the fresh shell. A per-build guard prevents any reload loop.

The result: after a deploy, the Home Screen app updates on its own within a second of opening,
instead of needing a manual cache clear. No behavior or API changes for content.
