## v1.5.2: agents can review, approve and schedule content

The agent write API now covers the full content lifecycle, so an AI agent or
script with an **Agent** token can do everything a human editor does, not just
create, update and publish.

- **Reviewer flow over the API.** New `POST /api/agent/:type/:id/review` records
  an approve or request-changes decision, and `GET /api/agent/:type/:id/reviews`
  returns the sign-off state and the workspace approval policy. Enough distinct
  approvals flip an entry to APPROVED; a request-changes sends it back to DRAFT
  and notifies the author, exactly like the studio. Reviews are attributed to the
  user who created the token.
- **Approve to Publish for live edits.** Editing an already published entry
  stages a pending draft while the public API keeps serving the live copy. New
  `POST /api/agent/:type/:id/approve-draft` promotes that draft, and
  `discard-draft` throws it away and keeps the live version.
- **Scheduling, documented.** Scheduling an auto-publish already worked via
  `PATCH status=SCHEDULED` with a `scheduledAt`; it is now a first-class,
  documented part of the agent API with a `schedule()` SDK helper.
- **SDK write helpers.** `@flowcms/client` gains `schedule`, `submitForReview`,
  `reviews`, `review`, `approveDraft` and `discardDraft` to match the routes.

Scope enforcement is unchanged: moving content into a sign-off state (approved,
scheduled or published) still requires the `content.publish` scope, so an
update-only token can draft and submit for review but can't bypass an approval
workflow. Backend and SDK only, no schema changes.
