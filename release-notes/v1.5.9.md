# v1.5.9: a tidier first run and steadier transactional emails

A small, polish-focused release covering first-run setup and the transactional
emails your install sends.

- **Setup asks for your name and workspace name up front.** The first-run wizard
  now requires both, so every install is properly identified the moment it comes
  online, no more unnamed "Administrator" accounts or a default workspace name to
  rename later. Existing installs are untouched.
- **Transactional emails hold their shape.** A long, unbreakable value (a stray
  URL, a long token) can no longer stretch a template past its frame on small
  screens; body text and detail panels wrap cleanly instead.
- **The SMTP test email has its own artwork.** Sending a test from Settings,
  Integrations, Email now shows a dedicated illustration instead of reusing the
  welcome art, so the test reads as exactly what it is.

No schema changes and no new endpoints. Studio and API.
