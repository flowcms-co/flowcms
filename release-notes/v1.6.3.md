# v1.6.3: updates now update everything, including the proxy config

Self-hosted installs run their app code from images, so upgrades have always
been one click. The proxy config was the exception: the Caddyfile is written to
the host once at install time, and a release that changed routing (like
v1.6.1's `/api/app-version` fix) silently left existing compose installs
behind unless someone SSH'd in and edited it by hand. That contradicts how
updates are supposed to work here, so this release closes the gap for good.

- **The updater now syncs release-managed deploy configs.** During every
  upgrade, after the new version is verified healthy, the updater refreshes the
  Caddyfile from the release and gracefully reloads Caddy, so routing fixes
  ship with the release like everything else.
- **Pending fixes apply on boot.** When the updater starts on a release it has
  not synced yet, it applies the config for the running version right away.
  Installs that upgraded before this feature existed pick up the v1.6.1
  routing fix automatically the first time their updater restarts after an
  upgrade, with no manual work.
- **Customized files are never touched.** Sync only replaces a Caddyfile it can
  prove is stock (a hash it wrote itself, a known shipped version, or the file
  as released at the running version). A hand-edited file is skipped with a
  clear log line telling you where to diff.
- **It can never break an upgrade.** Config sync is best-effort: any fetch or
  reload problem is recorded in the upgrade status and the upgrade still
  succeeds on the old routing.

Applies to docker-compose self-hosts (the all-in-one image used on managed
platforms already bakes its config in). No schema changes. Updater only.
