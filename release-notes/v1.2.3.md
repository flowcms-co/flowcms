## Highlights

This release brings the main editor's inline tools to every rich text field, gives each content type a clear Page type that sets its web address and SEO type in one choice, and keeps the Notifications and Chat pages current without a manual refresh.

## What's new

- Inline tools in every rich text field: select text inside a Rich text field to get a quick formatting bar (bold, italic, highlight, inline code, and links) plus an AI rewrite, the same as the main content editor. There is also an AI button on the field's toolbar. Before, these lived only on the main editor.
- A Page type for each content type: every content type now has a Page type, Blog, Service or Product, Home, or Static. One choice sets how that type's pages are addressed on your site and its structured-data (SEO) type. Blog and Service or Product pages live under their own section (for example /services/water-damage), Home is your site root, and Static pages like About us or Legal sit at the top level (for example /about-us).
- Fallback preview for unpublished pages: you can set a Fallback preview URL on each content type. A brand-new page with nothing published yet then previews on a representative page of that type, with the same on-page editing and the same notice you already get when borrowing a published page. The URL can be a full page address or a template with {slug}, {id}, {type}, or {locale} placeholders.

## Fixes and improvements

- Notifications and Chat stay current: both pages now refresh the moment you switch back to the tab, rather than waiting for the next poll. Browsers pause timers in background tabs, so returning to a tab no longer shows stale notifications or messages.
- General stability and performance improvements.
