## Fixes and improvements

- **Backups and restores no longer "reach into" the database container.** The updater used to run
  `pg_dump`/`psql` via `docker exec` on the running Postgres container. That worked, but it can't
  be routed through a least-privilege Docker socket proxy (exec needs a connection the proxy can't
  forward). The updater now runs `pg_dump`/`psql` from a short-lived Postgres client container that
  connects to the database over the network instead, an operation the proxy *can* forward. The
  database image is auto-detected so the client always matches the server version, and the full
  backup → restore round-trip is verified. Behavior is identical on the direct socket; this just
  makes the optional `secure-socket` profile fully functional end to end.

This unblocks running the updater with no direct Docker-socket access at all (the `secure-socket`
profile). It stays opt-in for now; once it's validated on a live upgrade it will become the default.
