# v1.6.2: a clean browser console on every screen

Open DevTools on the login screen and you'd see four failed requests; on a
managed deployment (Railway, Render) the dashboard added a 503 on every page
load. None of them broke anything, but red console noise erodes trust and
hides real problems. This release removes every one of them at the source.

- **The session probe no longer 401s.** `/api/auth/me` is the endpoint the
  studio calls to decide whether you're signed in, and "not signed in" is a
  valid answer, not an error. Anonymous requests now get `200 { user: null }`.
  Session validation is unchanged and every other endpoint stays guarded.
- **Signed-out screens make no authenticated calls.** Background jobs and
  workspace settings now load only after sign-in, so the login screen fires no
  request that could fail.
- **The upgrade status check runs only when it means something.** The studio
  polled `/api/system/upgrade/status` on every load to resume a mid-upgrade
  progress overlay. It now asks only when the same browser actually started an
  upgrade, so deployments without the in-app updater (managed platforms) never
  make the call, and the 503 disappears. Reloading mid-upgrade still resumes
  the progress overlay.

Pairs with v1.6.1, which fixed the `/api/app-version` 404 at the proxy. Update
to this release and the browser console is clean on the login screen, the
dashboard and everything in between, on compose and managed deployments alike.

No schema changes. Studio and API.
