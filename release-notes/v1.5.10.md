# v1.5.10: transactional emails render full-width on iOS Mail

A focused fix for how the emails your install sends look on Apple Mail.

- **No more shrunken emails on iPhone and iPad.** The email shell used a fixed
  600px width with a media-query fallback, and the hero illustration was sized at
  100%, which iOS Mail resolved to the image's full 1200px width and used to scale
  the whole message down to about half the screen. The container is now fluid (it
  fills the screen up to 600px) and the illustration carries a pixel width, so the
  welcome, invite, password-reset, notification, digest and SMTP-test emails fill
  the width on iOS just as they already did in other clients.

No schema changes and no new endpoints. API.
