# v1.6.1: silence the app-version 404 and make the shell healer work

Every install has been quietly logging a 404 on `/api/app-version` on each
page load and tab focus. The studio polls that route to detect a stale app
shell (mainly installed Home Screen apps on iOS) and reload it once after an
upgrade. The route lives on the studio's own Next server, but the reverse
proxy sent everything under `/api/` to the backend API, which has no such
route, so the check failed everywhere and the healer never ran.

- **Proxy routes `/api/app-version` to the studio.** An exact-path matcher now
  sits ahead of the general API route in all three shipped proxy configs: the
  single-domain Caddyfile, the all-in-one image used on Railway and Render, and
  the local smoke-test config.
- **Console noise is gone.** The repeated `Failed to load resource: 404
  (/api/app-version)` entries in the browser console disappear.
- **Stale-shell healing now works as designed.** After an upgrade, an installed
  app shell detects the new build id and refreshes itself once.

No schema changes, no new endpoints, no studio or API code changes. Deploy
config only; existing installs pick it up on their next image update.
