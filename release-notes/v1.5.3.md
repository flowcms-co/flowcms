# v1.5.3: the studio teaches itself

The studio now ships with an interactive guided tour, transactional email got
the full brand treatment with in-app code editing, and the character art
pipeline is ready for a fresh illustration pack.

- **Guided tour across every screen.** Thirteen chapters, one per area of the
  studio (Overview, Content, the block editor, SEO, the AI Optimizer, AI
  Tools, Assets, Chat, Team, Security, the content model, Integrations and
  Developers). Each chapter auto-plays the first time you visit its screen:
  an illustrated intro card, then spotlight steps that dim the page and point
  a caret at the exact control being explained. Full keyboard navigation and
  `prefers-reduced-motion` support.
- **The tour doubles as a setup checklist.** Steps that depend on an
  integration show a live connected / not-connected pill with a one-click
  Connect action: SMTP for team invites and password resets, an AI provider
  key for the generator and optimizer fixes, Search Console and GA4 for live
  SEO data. The compass button in the top bar tracks chapter progress and
  lists what still needs connecting.
- **Branded transactional emails.** All six default templates (welcome,
  invite, password reset, content published, notification, digest) are
  rebuilt on the FlowCMS email design system: brand lockup, illustration
  hero, one purple CTA, dark-mode and mobile support, inline-safe styles.
  The workspace name and studio URL are injected on every send, and the SMTP
  test email uses the same design so a test doubles as a preview.
- **Edit email templates like code.** Each template opens into a dark,
  line-numbered code editor with Split / Code / Preview views, a live
  preview rendered with sample data, variable chips that insert tokens at
  the cursor, an enable toggle and a reset-to-default action (new
  `DELETE /api/mail/templates/:key`). Saved customizations always win over
  the shipped defaults.
- **v2 character art pipeline.** Avatars and the matching dashboard hero
  illustrations now load from `/avatars/v2` and `/illustrations/v2` with
  automatic per-slot fallback to the classic packs, so a new art set can be
  rolled out one character at a time without a code change.

Studio and API; one new endpoint; no schema changes.
