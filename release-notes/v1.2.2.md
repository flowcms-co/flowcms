## Highlights

This release sharpens live editing: you can now add and remove items in accordion-style lists (like FAQs) right on the page, previews of unpublished pages are clearer, and the field list only shows content you can actually edit in place.

## What's new

- Edit accordion lists on the page: FAQ-style accordions (and any list built with expandable sections) now let you add, edit, and remove items directly in the live editor. The question inside an accordion header is now editable too, where before it couldn't be typed into. This works for any repeatable list, not just FAQs.
- Friendlier preview for unpublished pages: when you preview a page you have not published yet, FlowCMS borrows a published page's layout and now shows gentle placeholder hints (for example "Your title") in the empty fields instead of the other page's wording, so it is obvious what to fill in. The notice at the top is clearer and easier to read.
- Tidier field list: the field mapper and live editor now hide fields you cannot actually point at and edit on the page, image alt text, icon keys, and link or URL targets, on top of the SEO, meta, slug, and id fields already hidden. You see only the real on-page content. Images you can still swap stay editable.

## Fixes and improvements

- New Content now works from anywhere: clicking New Content while you are already in the editor starts a fresh, blank entry instead of doing nothing.
- Consistent naming for new content models: content types stay lowercase (they are part of your public web addresses, like /services/), while components and field keys use camelCase. This is now enforced everywhere new content is created, including imports. Your existing names are left exactly as they are.
- General stability and performance improvements.
