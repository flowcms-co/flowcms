## Fixes and improvements

- **The updater now updates itself.** Until now, a one-click upgrade refreshed the api and studio
  containers but never the updater sidecar, so any fix to the updater required a manual
  `docker compose pull updater` on the server, and an old updater would keep repeating its old
  behavior forever. Upgrades now self-update the updater too: on a successful upgrade it pulls the
  matching new updater image and recreates the updater in the background (via a short-lived helper
  container, since the updater can't recreate itself mid-run). No manual step, no drift.
- This pairs with the v1.3.6 disk fixes (prune on failed upgrades, pre-flight disk check,
  size-capped backups), so going forward the whole self-host upgrade path maintains itself.

> Note: this self-update only takes effect once you're running an updater that includes it. An
> existing install needs one final manual updater refresh (`docker compose pull updater &&
> docker compose up -d updater`) to pick up this version; after that, every future upgrade keeps
> the updater current automatically. New installs get it from the first upgrade.
