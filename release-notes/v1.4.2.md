## v1.4.2: mobile header and login polish

A small UX pass on the Studio shell for phones. No behavior or API changes.

- **Fixed header on mobile.** The top bar was a semi-transparent blurred strip that read as
  washed-out and let content show through. It is now a solid surface with a drop shadow, so it
  behaves like a proper fixed header and stays clearly separated from the page as you scroll.
  Desktop is unchanged (still a transparent, in-flow header).
- **Login no longer scrolls oddly.** The auth screens now use the dynamic viewport height
  (`100dvh`) with overflow-safe centering: the form centers when it fits and stays fully
  reachable (no clipped top) when it overflows a short phone viewport. This removes the stray
  scroll caused by the mobile browser toolbar.

Studio-only, CSS/markup changes; the split-screen desktop layout is untouched.
