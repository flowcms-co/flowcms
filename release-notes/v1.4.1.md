## v1.4.1 — least-privilege Docker access is now the default

The scoped, read-only `docker-socket-proxy` (introduced and made functional in v1.4.0) is now
**on by default**. The updater no longer receives the raw Docker socket; it talks to Docker only
through the proxy, which exposes just the handful of API calls the updater needs. A compromised
updater can no longer perform arbitrary host-root operations.

- No configuration required, new installs get this automatically.
- Existing installs pick it up on their next `install.sh` run (this is a compose change, so it
  arrives via the installer, not the in-app one-click upgrade).
- Backups, restores and upgrades all work through the proxy (validated on a live server before
  this default flip).
- To deliberately fall back to the direct socket (not recommended), set in `.env`:
  `UPDATER_DOCKER_HOST=` and `UPDATER_SOCKET=/var/run/docker.sock`.

This completes the production-hardening line: a self-host stack with log rotation, per-service
resource limits, dropped capabilities, healthchecks, graceful shutdown, network isolation, a
self-updating + self-pruning + disk-aware updater, and now least-privilege Docker access, all by
default, no manual steps.
