# v1.5.4: clearer updates on managed hosts

The Updates panel now speaks to how you actually deploy. If you self-host with
Docker Compose you still get the one-click, backed-up, auto-rollback upgrade. If
you run on a managed platform, it stops showing a dead end and tells you exactly
how to move versions there.

- **Names your host.** Flow CMS now detects when it's running on Railway or
  Render from the platform's own runtime environment and reports it on
  `/system/updates`. The Updates card badge reads "Managed by Railway" instead
  of a generic note.
- **Explains why the button is gone, and what to do instead.** On a managed
  deployment the in-app upgrade is intentionally disabled: the platform owns the
  container lifecycle and Flow CMS isn't given host Docker access, so it can't
  swap its own image in place. The card now says this plainly and shows
  step-by-step **redeploy** instructions for the detected host (Railway or
  Render), plus a link to the new Updating guide.
- **New "Updating Flow CMS" guide.** Covers the self-host one-click upgrade, the
  managed redeploy flow for Railway and Render, and a what-survives-an-update
  table (Postgres, S3 media, encrypted secrets). Cross-linked from the self-host
  guide.

Studio and API; no new endpoints; no schema changes.
