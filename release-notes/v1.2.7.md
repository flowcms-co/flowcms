## Highlights

This release reworks how reusable building blocks are modeled and named, fixes referenced components not showing their fields in the editor, and makes content keys and the importer friendlier. "Dynamic zone" is now the **Page builder**, and "Components" are now **Reusable components**, so the content model reads the way it actually works.

## What's new

- Clearer names for building blocks: a "Dynamic zone" is now called a **Page builder** (an ordered list of mixed sections an editor composes a page from), and the **Components** tab and field are now **Reusable components** (a named group of fields you embed or reference). The behavior is unchanged, only the wording, so existing models keep working.
- Friendlier field labels and descriptions in the editor: fields now show a readable label derived from their key (`ctaContent` reads as "Cta Content") plus a helpful description, even when none was set. You can still set your own label and description, and clicking "Edit label" or "Edit description" now pre-fills the derived value so you tweak it instead of starting from a blank box.
- See and open a referenced component from the content type: a Reusable component field now previews the fields it pulls in and gives you an "Edit" link that jumps straight to that component.
- Pick the import target from a list: when importing Markdown, CSV or JSON, "Import into content type" now offers a dropdown of your existing content types (you can still type a new api id to create one), with a compact, scrollable, theme-matched picker that opens upward when it sits near the bottom of the page.

## Fixes and improvements

- Referenced component fields now render in the editor: a Reusable component field that points at a library component (for example a shared "Hero" or "Quote") now shows that component's fields when editing an entry, instead of appearing empty. The same fix applies inside Page builder sections.
- Consistent, enforced field keys: field keys now show in camelCase as soon as a type opens (`Title` becomes `title`, `Cover image` becomes `coverImage`). When a key is renamed or camelCased, existing entries are migrated to the new keys in the same save, so authored content stays aligned and nothing is orphaned.
- Picking a component names the field for you: selecting a Reusable component while the field still has its placeholder name now adopts the component's name and refreshes its label and description.
- General stability and performance improvements.
