## v1.4.3: home-screen app name and shell caching

Two small fixes for the installable (Add to Home Screen) Studio app on iOS and Android.

- **Branded home-screen name.** The installed shortcut showed "Flow CMS". The PWA manifest
  name, short name, application name, and the iOS web-app title are now all "FlowCMS", so the
  home-screen app is branded consistently. Note: an icon that is already on your home screen
  keeps its old name; iOS only reads the name when you add it. Remove and re-add the icon once
  to pick up the new name.
- **No more stale shell.** The HTML shell (and the web manifest) are now served with
  `Cache-Control: no-cache, no-store, must-revalidate`. An iOS home-screen web-view keeps a
  private cache that "Clear Website Data" never clears, so a stale shell could pin old hashed
  chunks and show an outdated copy. Revalidating the shell makes new deploys land on launch.
  Content-hashed assets under `/_next/static` and `/_next/image`, plus the static brand, image,
  and email directories, are excluded, so asset caching is untouched.

Studio-only, configuration and metadata changes; no API or behavior changes.
