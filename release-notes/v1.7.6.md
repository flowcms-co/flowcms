# v1.7.6: leaner codebase, clusters fix

A housekeeping release. Around 2,700 lines of dead code are gone: 17
studio components nothing rendered anymore, the demo seed and stress
scripts, and four npm packages no code imported (swiper, millify,
scroll-lock and the Netlify build plugin). Self-hosted installs get
lighter too: docker-compose no longer starts the Meilisearch container
that nothing talked to, so every `docker compose up` saves the idle
memory it used to burn.

The string helpers that had drifted into six hand-written copies
(slugify, HTML stripping, HTML escaping) now live once in
@flowcms/shared, which also fixes a latent inconsistency where the same
title could produce different slugs depending on the code path. Live
preview clones entry data with structuredClone instead of a JSON
round-trip, so dates and rich values survive editing.

One real bug fix: the SEO Topical Clusters page crashed once live
Search Console data existed, because the live endpoint returns a
thinner cluster shape than the page expected. Live clusters are now
normalized with safe defaults and the page renders real data.

## Notes

Studio, API and packages. No migrations. The unused Meilisearch service
and its MEILI_* keys are gone from docker-compose and the example env
files; nothing read them, so existing installs need no action. Netlify
config is removed since it is not a supported deploy target.
