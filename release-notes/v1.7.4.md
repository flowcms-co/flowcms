# v1.7.4: no more browser popups

Native browser dialogs never matched the studio. Every remaining
window.alert — error notices in webhooks, the knowledge base, roles,
team, API keys, SCIM, IP policies, content scheduling and the content
model — now uses the same on-brand modal the rest of the studio does, in
a new single-button notice style. Confirmations already ran through the
branded dialog; its silent fallback to the browser's confirm is gone
too, so no code path can ever surface a native popup again.

## Notes

Studio only. No migrations, no config changes, no new environment
variables.
