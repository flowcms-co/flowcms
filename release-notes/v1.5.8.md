# v1.5.8: connect Resend or SendGrid, not just SMTP

Until now the Email integration only spoke SMTP. This release adds two hosted
email providers you can connect with nothing more than an API key, so getting
transactional email flowing no longer means hunting down SMTP host, port, and
credentials.

- **Resend and SendGrid, by API key.** Settings, Integrations, Email now shows a
  provider selector. Pick Resend or SendGrid, paste an API key and a from
  address, and you're connected. Both send over the provider's HTTPS API, so
  there's no SMTP host or port to configure, no nodemailer transport, and no new
  server dependency: the API calls go out through the built-in `fetch`.
- **SMTP is unchanged.** The existing SMTP option stays exactly as it was.
  Transactional email (invites, password resets, alerts, digests) is
  provider-agnostic and routes through whichever provider you connect, so
  templates and everything downstream keep working without a change.
- **Secrets are preserved on save.** Re-saving an already-connected provider
  without re-typing the key now keeps the stored one instead of overwriting it.
  This also fixes a case where updating just the from address could blank out the
  saved secret.
- **Test send covers all three.** The "send a test email" button uses whichever
  provider is currently connected.

No schema changes and no new endpoints. The existing `/mail/connect` endpoint
and Integration storage now carry a provider discriminator, with the API key or
password encrypted at rest as before. Studio and API.
