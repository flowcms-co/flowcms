## v1.5.1: delivery API dates fixed, live preview works out of the box

Two backend fixes for hosted installs.

- **Dates come back correctly from the delivery API.** On any install with a public
  media URL configured (MEDIA_PUBLIC_URL, or STUDIO_URL, which hosted deploys set),
  the media-absolutizing pass rebuilt each entry and collapsed its date fields to
  `{}`. `publishedAt`, `createdAt` and `updatedAt` now serialize properly again over
  both REST and GraphQL. Only plain objects are traversed; Date and other non-plain
  values pass through untouched.
- **Live preview is prefilled from your site URL.** Set `FRONTEND_URL` (your site
  origin, with `/{slug}` appended automatically) or `PREVIEW_URL` (a full template),
  and a fresh install points the workspace Preview URL at your real site, so live
  preview renders your pages instead of the demo. A Preview URL you set in
  Settings → System is never overwritten.

Backend only, no schema changes. If your delivery responses had empty date fields,
or live preview showed the demo, update to v1.5.1.
